# 🎯 어그로필터 미션 수행 규칙 (Hub AI's Integration Bible)
- **작성/관리 주체**: **허브 AI (안티그래비티)** 전용 기록지.
- **필독 (개별앱 AI)**: 
  > **경고**: 개별 앱 AI(윈드서퍼 등)는 본 문서에 일지나 개인 의견을 작성하지 마십시오. 
  > 본 문서는 허브 AI가 연동 과정을 정밀 기록하여 향후 다른 패밀리 앱 연동 시 '절대 기준'으로 삼기 위한 기술서입니다.
- **업데이트 시점**: 각 미션 단계 완료 시 허브 AI가 즉시 기록함.

---

## 1. 연동 마일스톤 및 기술 기록 (Mission Records)

### [미션 1] 프로필 중앙화 (완료)
- **핵심 기술**: 허브 `family_users`와 앱 `t_users` 간의 Email 기반 매핑.
- **바이블**: 신규 앱 연동 시 프로필 이미지는 반드시 Base64 리사이징(200px) 정책을 강제해야 함.

### [미션 2] 데이터 마이그레이션 및 UID 매핑 (완료)
- **핵심 기술**: 어그로필터 레거시 데이터(409건)를 허브 UUID로 전환.
- **바이블**: 마이그레이션 시 반드시 모든 관련 테이블(analyses, subscriptions, history 등)을 동시에 매핑해야 함. 일부만 하면 프론트에서 데이터가 안 보이는 사고 발생.

### [미션 3] family_uid 폐기 및 UUID 단일화 (완료 — 2026-04-25)
- **사고 원인**: 허브 AI가 `family_users`에 `id`(UUID)와 `family_uid`(mfn-xxx) 두 개의 식별자를 만들어 이원화 사고 발생.
- **해결**: 
  1. 어그로필터 DB의 모든 `mfn-xxx` 값을 허브 UUID로 일괄 재매핑.
  2. 허브 DB에서 `family_uid` 컬럼 완전 폐기.
  3. JWT, SDK, 프론트엔드 localStorage 전부 UUID 기반으로 전환.
- **바이블 (절대 교훈)**: 
  - **유저 식별자는 반드시 1개만 존재해야 한다.** "사람이 읽기 쉬운 ID"는 핑계에 불과하다.
  - 향후 2호, 3호 앱에서도 절대 커스텀 ID를 추가하지 말 것.

### [미션 4] 중앙 과금 엔진 구축 (진행 중)
- **계획**: `family_coin_balances` + `family_coin_history` + `family_aggro_video_pricing` 테이블 신설.
- **핵심 원칙**: 앱은 자체 지갑을 가질 수 없으며, 코인 차감/충전은 반드시 허브 API를 통해서만 처리.
- **검수 포인트**: `request_id` 멱등성 보장, `balance_after` 스냅샷 기록.

## 2. 라이브러리(SDK) 표준화
- `소통관/라이브러리/merlin-hub-sdk`를 모든 앱의 표준으로 사용.
- SDK 수정은 반드시 허브 라이브러리 선(先)수정 후 앱 전파 원칙 고수.
- SDK의 `verifyOTP` 응답에서 `userId`(UUID)를 `localStorage.merlin_user_id`에 저장.

## 3. 개별 앱 유저 테이블 표준 (t_users 다이어트)
- PK는 반드시 허브 UUID (`user_id UUID NOT NULL PRIMARY KEY`).
- 허브와 중복되는 필드(이메일, 닉네임, 코인) 절대 금지.
- 앱 고유 서비스 데이터(티어, 랭킹, 알림 설정 등)만 보관.
