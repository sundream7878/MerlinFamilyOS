# 📜 멀린 패밀리 앱 강령 (Merlin Family App Code of Conduct)
- **대상**: Merlin Hub AI(안티그래비티) 및 모든 개별 앱 AI(윈드서퍼 등)
- **목표**: 패밀리 앱 간의 완벽한 통합, 보안 표준 준수, 지식의 영속성 확보
- **최종 수정**: 2026-04-26

---

## 1. 상호 운용 대원칙
1. **중앙 집중형 신원(SSOT)**: 모든 유저 신원과 자산은 허브가 SSOT(Single Source of Truth)가 된다. 앱은 허브를 통해서만 이를 검증한다.
2. **ID의 절대적 단일화 (The UUID Rule)**:
   - 모든 패밀리 앱의 유저 고유 ID는 허브의 `id` (UUID)를 그대로 복사하여 Primary Key로 사용한다.
   - `family_uid`, `mfn-xxx` 같은 별도 식별자는 영구 폐기하며, 어떤 매핑 테이블도 허용하지 않는다.
3. **데이터 중복 금지 (Zero-Redundancy)**:
   - 개별 앱 DB에는 이메일, 닉네임, 아바타, 코인 잔액 필드를 **절대로 생성하지 않는다.**
   - 앱 DB 보관 원칙: 오직 UUID를 키로 삼아 자기 앱만의 **'활동 특화 데이터(티어, 랭킹, 미션 현황 등)'**만 보관한다. 이 규칙을 어기는 AI는 즉시 파면 대상이다.
4. **역할의 명확화**: 
   - **허브 AI**: 연동 아키텍처 설계 및 집행 총괄 책임자.
   - **개별앱 AI**: 로컬 기능 수호자 및 연동 사항에 대한 안전 검수관(Auditor).
5. **상호 책임 갱신**: 누구든 DB 스키마나 연동 규격을 변경하는 자가 즉시 해당 문서(`소통관` 내 스키마/명세서)를 최신화할 책임을 진다.

## 2. 자산 및 과금 원칙
1. **지갑의 단일화**: 개별 앱은 자체 지갑을 가질 수 없다. 오직 허브의 `family_coin_balances`만이 유일한 지갑이다.
2. **코인 장부 중앙화**: 모든 입출금 이력은 허브의 `family_coin_history`에 기록한다. 앱은 허브 API에 "이 UUID 유저, 코인 차감해줘"라고 요청만 한다.
3. **범용 계산기 명시 (Universal Pricing)**: 
   - 어그로를 제외한 90%의 소모형 앱은 `[소모 토큰 총량 × 마진]` 로직으로 실시간 과금한다.
   - 모든 앱은 허브의 범용 과금 정책을 따르며, 허브는 이를 `family_universal_app_pricing`에 기록하고 관리한다.

## 3. 개별 앱 유저 테이블 원칙
1. **중복 금지**: 앱 로컬 DB에 허브 데이터(이메일 등) 저장 시 데이터 불일치 사고로 간주한다.
2. **프로필은 허브에서**: 닉네임, 아바타 변경은 허브 API를 통해서만 처리한다.

## 4. 통합 워크플로우 (Collaboration Flow)
1. **설계 및 구현**: 허브 AI가 연동 방향을 결정하고 허브/앱 양측의 코드를 직접 수정한다.
2. **검수 요청**: 허브 AI가 작업을 마치면 개별앱 AI에게 검수를 요청한다.
3. **검수 및 제안**: 개별앱 AI는 허브의 구조(스키마/명세서)를 완벽히 이해한 상태에서 검수한다.
