# 🏛️ 허브앱 기능 및 기술 명세서 (Hub Technical Spec)
- **작성 주체**: **허브 AI (안티그래비티)** 가 관리하며, 변경 시 즉시 최신화한다.
- **활용 용도**: 개별 앱 AI(윈드서퍼 등)가 허브의 구조와 기술 스택을 이해하고 연동 정합성을 검수하기 위한 기준서.

---

## 1. 기술 스택 (Technical Stack)
- **Framework**: Node.js (Express) with TypeScript
- **Database**: Supabase (PostgreSQL) - RLS(Row Level Security) 적용
- **Auth**: 
  - JWT (JSON Web Token) - 클라이언트 세션 관리
  - Email OTP (Resend API 연동) - 무비밀번호 로그인
- **Infrastructure**: Vercel (Deployment) / Local Node Server
- **Communication**: 
  - RESTful API (JSON)
  - S2S (Server to Server) 인증: `X-App-Secret` 헤더 기반 검증

## 2. 핵심 서비스 로직
### [인증 서비스 - AuthService]
- `verifyOTP`: 이메일 인증 성공 시 유저 정보를 `family_users`에 **UPSERT**.
- `generateToken`: 유저의 `id`, `email`, `family_uid`를 포함한 JWT 발급.
- `getProfile`: 세션 기반 닉네임/아바타(Base64) 제공 및 앱 간 프로필 동기화.

### [지갑 서비스 - WalletService]
- `getBalances`: 유저의 통합 자산(`f_credits`) 조회.
- `processTransaction`: 멱등성(`request_id`) 기반 결제/충전 처리. 
  - `family_wallet_transactions` 기록 및 `family_wallet_balances` 실시간 업데이트.

## 3. 연동 보안 정책 (Security Policy)
- **App Secret**: 각 앱별 고유 `client_secret`을 발급하여 허브 API 호출 시 무결성 검증.
- **CORS**: 허용된 도메인(어그로필터 등)만 허브 API 접근 가능.
- **Rate Limit**: 무차별 대입(Brute-force) 방지를 위한 OTP 요청 횟수 제한.

## 4. 라이브러리 및 SDK
- **Merlin Hub SDK**: 개별 앱용 공용 라이브러리 (위치: `소통관/라이브러리/merlin-hub-sdk`)
- **주요 모듈**: `auth.ts`, `wallet.ts`, `client.ts`

---
> **검수관(윈드서퍼) 필독**: 허브의 `UPSERT` 로직과 `mfn-xxx` 번호 체계는 패밀리 앱 통합의 근간이다. 어그로필터 연동 시 이 규격을 어기지 않도록 주의 깊게 확인할 것.
