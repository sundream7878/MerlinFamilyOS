# 🏛️ 허브앱 기능 및 기술 명세서 (Hub Technical Spec)
- **작성 주체**: **허브 AI (안티그래비티)** 가 관리하며, 변경 시 즉시 최신화한다.
- **활용 용도**: 개별 앱 AI(윈드서퍼 등)가 허브의 구조와 기술 스택을 이해하고 연동 정합성을 검수하기 위한 기준서.
- **최종 수정**: 2026-04-25

---

## 1. 기술 스택 (Technical Stack)
- **Framework**: Node.js (Express) with TypeScript
- **Database**: Supabase (PostgreSQL) - RLS(Row Level Security) 적용
- **Auth**: 
  - JWT (JSON Web Token) - 클라이언트 세션 관리
  - Email OTP (Resend API 연동) - 무비밀번호 로그인
- **Communication**: 
  - RESTful API (JSON)
  - S2S (Server to Server) 인증: `X-App-Secret` 헤더 기반 검증

## 2. 유저 식별 체계 (절대 원칙)
- **UUID 단일화**: `family_users.id` (UUID)가 유일한 유저 식별자.
- **폐기됨**: `family_uid` (mfn-xxx) — 2026-04-25부로 완전 폐기. 코드 및 DB에서 제거 완료.
- **JWT 페이로드**: `{ email, userId(UUID), iat }`
- **앱 연동 시**: 앱은 허브 UUID를 그대로 자신의 유저 테이블 PK로 사용한다.

## 3. 핵심 서비스 로직
### [인증 서비스 - AuthService]
- `verifyOTP`: 이메일 인증 성공 시 유저 정보를 `family_users`에 **UPSERT**. UUID 자동 생성.
- `generateToken`: 유저의 `userId(UUID)`, `email`을 포함한 JWT 발급.
- `getProfile`: 세션 기반 닉네임/아바타(Base64) 제공.

### [코인 서비스 - WalletService (→ CoinService로 전환 예정)]
- `getBalance`: UUID로 직접 `family_coin_balances` 조회. (중간 변환 없음)
- `processTransaction`: 멱등성(`request_id`) 기반 코인 차감/충전 처리.

## 4. 중앙 과금 엔진 (신규)
### [테이블 구조]
- **`family_coin_balances`**: 유저별 코인 잔액 (UUID → balance)
- **`family_coin_history`**: 모든 앱의 코인 입출금 장부 (transaction_type으로 앱 구분)
- **`family_aggro_video_pricing`**: 어그로필터 영상별 단가표 (허브에서 일괄 관리)

### [과금 플로우]
1. 앱이 허브에 "이 UUID 유저, 코인 차감해줘" 요청
2. 허브가 잔액 확인 → 차감 → `family_coin_history` 기록 → 잔액 갱신
3. 결과를 앱에 반환

### [보상 정책]
- EARN_WELCOME: 첫 가입 +1,000C (1회 한정)
- EARN_REFERRAL: URL 추천 +500C (초대자)
- EARN_ONBOARDING: 앱별 미션 +1,000C

## 5. 연동 보안 정책
- **App Secret**: 각 앱별 고유 `client_secret` 발급, API 호출 시 검증.
- **CORS**: 허용된 도메인만 허브 API 접근 가능.

---
> **검수관(윈드서퍼) 필독**: 허브의 UUID 단일 체계와 코인 중앙화 원칙은 패밀리 앱 통합의 근간이다. 어그로필터 연동 시 이 규격을 어기지 않도록 주의 깊게 확인할 것.
